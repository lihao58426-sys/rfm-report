# RFM 客户价值分析系统 — 技术文档

> 版本：V2.0
> 日期：2026年7月10日
> 用途：银豹 CSV → RFM 8类分群 + 客户生命周期 + CLV + 留存曲线 → 浏览器看报告

---

## 一、项目概述

两模式：静态 HTML 报告（脱敏样例展示）+ FastAPI 在线分析（上传真实 CSV 自动出报告）。支持多文件合并、数据清洗、佚名分离、RFM 分群、生命周期阶段、CLV 估算、同批次留存曲线、月度趋势、ECharts 可视化。

---

## 二、技术栈

| 层面 | 技术 | 说明 |
|------|------|------|
| 后端 | FastAPI 0.139 | 文件上传 + 分析接口 |
| 数据库 | 无（纯计算引擎） | CSV 读入 → 内存计算 → 输出结果 |
| 模板 | Jinja2（原生 Environment） | 绕过 Starlette Jinja2Templates 缓存 Bug |
| 前端 | 原生 HTML + CSS + ECharts 5.5 | 柱状图/饼图/折线图 |
| 静态版 | Vanilla JS + CSS | 脱敏样例，纯前端无依赖 |
| CSV 解析 | Python csv 模块 | 标准库，零依赖 |

## 三、系统架构

```
浏览器
  │
  ├─ GET  /         → upload.html（上传页）
  │
  └─ POST /report   → 4 个 CSV → analysis.py → report.html（含 ECharts）
                          │
              ┌───────────┼───────────┐
              ▼           ▼           ▼
          数据清洗     RFM 计算    生命周期/CLV/留存
```

### 数据清洗流程

```
4 个 CSV → 合并去重表头 → 过滤退款/空会员/异常金额
         → "张三（138...）"拆为 name+phone
         → "-"归入佚名池
         → 按日期排序
         → 进入分析
```

## 四、分析模块（analysis.py）

### 4.1 RFM 计算

按手机号分组 → R=(今天-最后消费日期)、F=消费次数、M=实收金额合计 → 跟均值比 → 8 类分群

### 4.2 客户生命周期

R 阈值 ≤30/30-90/>90 天 + F 阈值 ≤2/3-5/≥6 次 → 新客/成长/成熟/休眠/流失

### 4.3 CLV 估算

```
CLV = 每次均消 × 年均次数 × 预计剩余年数
预计生命周期 = 5 年（卡丁车客户的典型周期）
```

### 4.4 留存曲线

按首次到店月份分批次 → 追踪每批次每月留存率 → M0~M12 矩阵

## 七、使用方式

## 八、开发踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| Jinja2Templates 500 | Python 3.14 + Starlette 缓存不兼容 | 改用 `jinja2.Environment` 直接加载 |
| StaticFiles 冲突 | 整个目录挂在 /static 抢走了模板路径 | 只挂 `static/` 子目录 |
| 佚名收入少 3 万 | `total_revenue` 只算了可识别会员 | 加 `+ anonymous["revenue"]` |
| 端口 8000 冲突 | 两个 FastAPI 抢端口 | RFM 改为 8001 |
| ECharts 需要外网 | CDN 不可用（内网场景） | 下载 echarts.min.js 存本地 |
